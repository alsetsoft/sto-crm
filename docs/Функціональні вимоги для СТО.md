- Функціональні модулі

 План (Наряди-замовлення) — Список усіх нарядів зі статусами, фільтрами за статусом,
пошуком за номером, клієнтом або авто.
 Створення наряду — Майстер у 3 кроки: вибір/створення клієнта → вибір або створення
авто (з авто-заповненням марки, моделі, року і інших даних через API) → опис заявки
клієнта.
 Картка наряду — Workflow-степер, дані клієнта й авто, призначення виконавців з числа
працівників, блок приймання (фото авто, примітки, електронний підпис клієнта), список
робіт (з довідника послуг або вручну), список деталей (зі складу з автоматичним
списанням або вручну), блок «Проблеми», розрахунок суми, часткова/повна оплата.
Оплати готівка, безнал.
 Проблеми — Глобальний реєстр проблем по всіх нарядах. Кожна проблема має рівень
критичності (низька / середня / висока) і статус (відкрита / закрита). Передбачено фільтри
за статусом і критичністю, пошук за описом, клієнтом, авто або номером наряду.
 Клієнти — Список клієнтів з пошуком за ПІБ або телефоном. Картка клієнта містить
контактні дані, перелік авто та повну історію візитів. Підтримується додавання,
редагування й видалення.
 Авто — Каталог авто з пошуком за маркою, моделлю, VIN, державним номером або
власником. Картка авто включає марку, модель, рік, VIN, номер двигуна, держ. номер,
пробіг, тип КПП, привід, кузов, потужність, коментар.
 Склад — Позиції з артикулом, штрих-кодом, категорією, підкатегорією, місцем зберігання.
Облік залишків, мінімального запасу, рекомендованого запасу, цін закупки (остання та
середня) і продажу. Підсвічування позицій нижче мінімуму та в мінусі. Архівування
позицій, підрахунок загальної вартості складу.
 Послуги — Довідник робіт із ціною та нормо-годинами. Використовується для швидкого
додавання робіт у наряд.

 Звітність (Працівники) — Список працівників, місячна ставка, відсоток від виконаних
робіт, нарахування за поточний місяць, виплати (зарплата / аванс / премія), баланс до
виплати..

Додаткові функціональні вимоги
- План робіт
 У системі має бути розділ «План».
 Розділ «Наряд-замовлення» потрібно перейменувати на «План».
 У розділі «План» має відображатися тижневий план робіт для всіх працівників.
 Кожен наряд у плані має відображатися окремим блоком.
 Фото по наряду мають додаватися або передаватися через Telegram.
- Кольорове відображення статусів
 Колір має застосовуватися до всього блоку наряду, а не тільки до тексту або
бейджа статусу.
 Для статусів використовуються такі кольори:
Статус Колір блоку
## Запис Сірий
## Прийнято Синій
В роботі Помаранчевий / легкий жовтий
## Готово Зелений
## Розраховано Фіолетовий
## Проблема Червоний
- Назви розділів
 Розділ «Наряд-замовлення» має називатися «План».
 Розділ «Замовники» має називатися «Клієнти».
## 4. Пошук
 У системі має бути можливість пошуку нарядів / авто за:
 VIN-кодом;
 державним номером авто.

## 5. Склад
 У модулі «Склад» потрібно додати можливість прикріплювати фото до позиції складу.
 Для кожної позиції складу мають бути поля:
 вхідна ціна;
 вихідна ціна.
 На основі вхідної та вихідної ціни має розраховуватися маржа на запчастинах.
## 6. Проблеми
 У системі має бути окрема вкладка «Проблеми».
 Проблема може створюватися тільки тоді, коли авто перебуває у статусі «В роботі».
 Проблема має бути прив’язана до конкретного наряду.
 Менеджер має мати можливість змінити статус проблеми на «Вирішено».
 Для цього в інтерфейсі має бути дія / кнопка «Вирішити проблему».
- Працівники та звітність
 У розділі «Працівники» має бути блок або вкладка «Звітність».
 Для кожного працівника має бути можливість встановлювати індивідуальний відсоток
від виконаних робіт.
 У звітності мають бути фільтри за періодом:
 тиждень;
 місяць;
 рік;
 довільний період «Від — До».
 У звітності мають відображатися наряди, у яких працівник був виконавцем.
 Для кожного працівника потрібно показувати:
 перелік його нарядів;
 суму робіт;
 встановлений відсоток;
 суму нарахування.
- Видимість фінансової інформації для працівників
 Працівники не повинні бачити внутрішні фінансові дані по наряду.
 Працівник бачить тільки кінцеву вартість наряду.
 Виняток — наряди у статусі «Розраховано», для яких видимість фінансової
інформації визначається роллю користувача.
## 9. Документи
 У системі має бути можливість формувати акт приймання авто.
 Акт приймання має бути доступний для друку.
 У системі має бути можливість формувати акт виконаних робіт.
 Акт виконаних робіт має бути доступний для перегляду та друку.
- Міграція даних і почати розробку із Складу
